現在の会話を要約した引き継ぎドキュメントを作成し、別のエージェントが作業を継続できるようにする。

## 保存先

OS の一時ディレクトリ（例: `$TMPDIR` / `/tmp` など）に保存する。現在のワークスペースには保存しない。

## 記載する内容

- 何に取り組んでいたか（目的・背景・現在地）
- これまでに決めたこと・試したこと・その結果
- 未解決の課題、次にやるべきこと
- 関連するファイルパス・コマンド・参照先
- 「推奨 skill」セクションを設け、次のエージェントが呼び出すべき skill を提案する

## 方針

- 他の成果物（PRD、plan、ADR、issue、commit、diff など）に既に記録されている内容を重複させない。パスや URL で参照するに留める。
- API キー・パスワード・個人を特定できる情報などの機微情報は伏せる（redact する）。
- 引数が渡された場合は、それを「次のセッションが何に注力するか」の説明として扱い、その観点に合わせてドキュメントを調整する。
